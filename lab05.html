Lab 5: Vibe Coding

Enhancing My Portfolio with AI Assistance

Lab Requirements

Assignment: Explore AI-assisted coding through practical experiments and critical reflection.

  • Complete 2+ experiments: enhance portfolio, generate Python code, or debug with AI
  • Document code with honest comments about what you understand vs. still learning
  • Write 400-600 word reflection on when AI helps, when it doesn't, and personal guidelines for use
  • Key principle: You must be able to explain and modify any code you use

Lab 5 Reflection

For the Vibe Coding lab, I decided to both enhance my portfolio and debug an error that came with that enhancement. Although it does not look like I made large changes, I found this more beneficial to my learning because I got to implement smaller, easier-to-understand chunks of code rather than letting my imagination run wild, but having the AI do a bunch of things that I do not understand.

Adding Dark Mode

First, I took inspiration from one of the lab suggestions and planned to add a dark mode feature to the website. I had this option on the portfolio I made last semester and really enjoyed it, so I thought it would be a good addition here too. The initial prompt I asked was, "I am looking to add a dark mode feature to my portfolio and would like assistance. Would I primarily make this change on the CSS page?" The AI was helpful here as it gave me a breakdown, by percentage, of where all of the work to implement dark mode would be happening. It said that 60% would be CSS, 20% would be HTML, and 20% would be JavaScript. Even understanding how all three languages were working together here, rather than just seeing the code it provided, helped my learning process so much more. The AI then gave me step-by-step instructions on where to add each chunk of code within my pages. I appreciated going in and doing this myself rather than letting the AI do it and simply hitting the "keep" changes button because it allowed me to learn through a little bit of trial and error with the specific code placement. However, I will say that I added all of the changes myself on the first HTML page, but let the AI go in and add all of the same changes on the other pages. I thought that was okay because I already learned what to do, but it was a huge time-saver in the means of redundancy. Overall, in this section, I learned that the CSS added the dark colors and smooth transitions, the HTML added the toggle button and navigation, and the JavaScript added the toggle function and localStorage to remember the choice.

Debugging Accessibility Issues

After adding the dark mode button, I noticed an accessibility error in that whenever I shrank the preview screen, the button would appear on top of the already established navigation buttons. This is where I would say the AI was a little problematic because I would not expect it to cause an error that seems so simple to avoid, but I am also glad it did because it led to another learning opportunity. After asking a few simple questions to fix the error, I noticed I was not completely understanding what all of the code meant so I asked, "what does this part specifically do: .dark-mode-toggle:hover {transform: scale(1.05); /* Remove translateY from hover */" Here I learned that whenever the user hovers over the button on a mobile device, it grows 5% larger to create a subtle pop effect, and this is to give the user feedback that the button is working and interactive. That is something that I never would have thought of, but I think is such a great subtle addition, and I am so glad I can understand what that part of the code is doing now.

Additional Refinements

Those were the main areas I targeted for this lab, but I also went in and changed a few other parts. For example, I really did not like the colors the AI provided for the dark mode feature, so I went in and changed all of those manually. I ran into a slight roadblock when doing that because I initially could not see any changes on the preview when I was changing the colors. This led me to ask AI why that was happening, and it told me that I actually had a duplicate "body.dark-mode" section in my CSS. I think that is a great example of how AI can find a human error that is so easy for us to overlook, in a matter of seconds.

Reflections on AI-Assisted Learning

Overall, I had a good time with this assignment, and I appreciate labs like this where I get to take time to meaningfully learn with AI. I do think there can be tradeoffs to using a lot of AI assistance to code, especially in a class like ours, where most people do not have coding experience. I think that sometimes, because our portfolios are large, interactive, and have a lot of components, it is easy for us to overlook things and not take time to understand what the code is doing, just because there is so much of it. Because of that, one might not even know where to go to make certain changes, and will ask AI to do it just because it is easier. I am trying to get better about this myself because, since I am starting to code and use new languages in so many of my classes, I know how important it is to truly understand what I am doing. That is why I decided to tackle what might not look like a very hard task for this lab. Although it might look simple from an outside perspective, I actually took the time to understand and teach myself what each line is doing, and I think that pushed me more than tackling something bigger would have.
